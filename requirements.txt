Шахматный симулятор (ООП-версия)
main.py — основной файл программы, содержит все классы и точку входа

1) Структура классов (иерархия)
Базовый класс Piece (абстрактный)
Атрибуты: color (цвет), position (позиция)
Методы: get_symbol(), get_possible_moves()
Назначение: определяет общий интерфейс для всех игровых фигур

2) Классы шахматных фигур (наследуют Piece):
Pawn (пешка), Rook (ладья), Knight (конь), Bishop (слон), Queen (ферзь), King (король).
Каждый класс реализует свою логику движения в методе get_possible_moves()

3) Класс Checker (наследует Piece)
Назначение: фигура для игры в шашки
Особенность: ходит только по диагонали, реализует взятие прыжком

4) Базовый класс Board
Атрибуты: _grid (игровое поле 8x8)
Методы: get_piece(), set_piece(), is_within_bounds(), is_check(), get_threatened_positions(), display()
Назначение: инкапсулирует состояние игровой доски

5) Класс ChessBoard (наследует Board)
Назначение: доска для шахмат с классической расстановкой фигур
Класс CheckersBoard (наследует Board)
Назначение: доска для шашек, фигуры расставляются на тёмные клетки в три ряда

6) Класс Move
Атрибуты: piece (фигура), start (начальная позиция), end (конечная позиция), captured (съеденная фигура)
Назначение: хранит информацию о ходе для возможности отката

7) Класс ChessGame (контроллер игры)
Атрибуты: board (доска), history (история ходов), current_turn (текущий игрок), is_game_over (флаг окончания игры)
Методы: make_move(), undo_move(), get_valid_moves_for_piece(), show_hints(), run()
Назначение: управляет игровым процессом, валидацией ходов и очередностью

8) Класс CheckersGame (наследует ChessGame)
Назначение: контроллер для игры в шашки
Особенность: переопределяет логику валидации ходов (без проверки шаха) и условия победы

Ключевые принципы ООП в проекте
Инкапсуляция
Состояние доски скрыто в приватном атрибуте _grid. Доступ к клеткам осуществляется только через методы get_piece() и set_piece(). Фигуры хранят свои координаты внутри себя и обновляют их через setter.
Наследование
Иерархия фигур: все фигуры наследуют от абстрактного класса Piece.
Иерархия досок: ChessBoard и CheckersBoard наследуют от Board.
Иерархия игр: CheckersGame наследует от ChessGame, переиспользуя большую часть логики.
Полиморфизм
Метод get_possible_moves() реализован по-разному для каждого типа фигуры. Контроллер игры вызывает этот метод, не зная конкретного типа фигуры, что позволяет легко добавлять новые типы.
Абстракция
Базовый класс Piece определяет обязательный интерфейс (get_symbol, get_possible_moves), но не реализует логику движения. Это гарантирует, что любая новая фигура будет иметь необходимые методы.
